# Make Plan

创建详细的分阶段实施计划。用于规划功能开发、多步骤任务、复杂项目。

## When to Use

- 用户说"帮我规划"、"怎么实现"、"做个方案"
- 复杂任务需要拆解成多个阶段
- 在执行 `do` 技能之前

## How It Works

### 委派模型

用 subagent 做**事实收集**（文档、示例、API 签名、grep 结果）。
**综合和计划撰写**留给主 agent（阶段边界、任务框架、最终措辞）。

### Subagent 报告契约（强制）

每个 subagent 必须返回：
1. **Sources** — 查了哪些文件/URL，读了什么
2. **Findings** — 具体发现（API 名称/签名、文件路径）
3. **Copy-ready snippets** — 可复制的代码片段位置
4. **Confidence + gaps** — 置信度 + 已知空白

如果 subagent 报告结论但没有来源 → 拒绝并重新派遣。

## Plan Structure

### Phase 0: 文档发现（永远第一步）

在规划实施之前，先派 subagent 去：
1. 搜索并阅读相关文档、示例、现有模式
2. 识别实际可用的 API、方法和签名（不是假设的）
3. 创建"可用 API"列表，引用具体文档来源
4. 标注反模式（不存在的方法、已废弃的参数）

### 每个实施阶段必须包含

1. **What** — 要实现什么（引导从文档复制，不是凭空发明）
2. **References** — 具体文件/行号的文档引用
3. **Verification** — 如何证明这个阶段成功（测试、grep 检查）
4. **Anti-patterns** — 不要做什么

### 最终阶段: 验证

1. 验证所有实现与文档匹配
2. 检查反模式（grep 已知坏模式）
3. 运行测试确认功能

## Anti-Patterns to Prevent

- ❌ 发明"应该存在"的 API 方法
- ❌ 添加文档中没有的参数
- ❌ 跳过验证步骤
- ❌ 不检查示例就假设结构
